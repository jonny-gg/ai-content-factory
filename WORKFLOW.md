# AI Content Factory Workflow

## 默认工作模式

从现在开始，针对 Jonny 的“看看状态 / 最新进展 / 当前仓库怎么样 / 进度如何”这类询问，默认执行模式不是“只汇报”，而是：

**先检查当前状态，再顺手推进一小段代码，然后一起汇报。**

也就是说，每次状态询问都尽量包含这三部分：

1. **当前状态**
   - 当前分支/未提交变更/最近提交
   - 当前已完成阶段
   - 当前阻塞点

2. **本轮主动修改**
   - 至少完成一个最小可验证改动，优先级：
     - 修复明显 bug
     - 打通下一段 pipeline
     - 补充 CLI 参数/入口
     - 补充类型/契约
     - 更新过时文档
   - 如果不适合直接改代码，至少补一个明确 TODO 或设计占位文件

3. **验证结果**
   - 运行 typecheck / test / dry-run / CLI help 等至少一种验证
   - 说明改动影响范围

## 优先级规则

当用户只是问“状态”时，默认优先做这类推进：

1. render / subtitles / ffmpeg 相关最后一公里
2. assets pipeline 稳定性
3. story pipeline 契约稳定性
4. CLI 易用性
5. 文档纠偏（只在代码不适合动时）

## 例外情况

以下情况可以只汇报、不改代码：

- 用户明确说“先别动代码”
- 当前需要先确认方向/需求
- 改动会涉及破坏性重构
- 环境缺失导致无法安全推进

如果只汇报，必须明确说明为什么本轮没有改代码。

## 目标

把互动模式从：

`你问状态 -> 我口头汇报`

改成：

`你问状态 -> 我检查状态 -> 我顺手推进代码 -> 我汇报结果`
